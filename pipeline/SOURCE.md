# SOURCE.md — выбор источника НПА (Роль 3, spike дня 1)

> Решение принято 6 июля 2026. Проверено живыми запросами, а не по описаниям.

## Решение: ИПС «Законодательство России» (pravo.gov.ru)

Официальная государственная информационно-правовая система (оператор — ФСО России).
Берём **консолидированный текст в действующей редакции** двухшаговой схемой
(реализация — `pipeline/fetch.py`):

```
1) http://pravo.gov.ru/proxy/ips/?docbody=&nd={ND}
   -> в HTML есть выпадашка редакций: <option value='{rdk},{ND}'>N - от DD.MM.YYYY № ...-ФЗ</option>
   -> берём максимальный rdk (последняя редакция) и её дату
2) http://pravo.gov.ru/proxy/ips/?savertf=&nd={ND}&page=all&rdk={rdk}
   -> MHTML-контейнер (multipart/related), внутри text/html со ВСЕМ текстом документа
```

- `nd` — id документа в ИПС (ТК РФ: `102074279`); кодировка ответов — **windows-1251**.

### ⚠️ Грабли, найденные при проверке (не наступать повторно)

1. **`rdk=0` — это ПЕРВОНАЧАЛЬНАЯ редакция, а не действующая.** Для ТК РФ это текст
   2001 года: без ст. 64.1, без дистанционной работы (гл. 49.1), без ст. 351.7.
   Для юридического бота это катастрофа — всегда брать последний `rdk` из списка редакций.
2. **Основной endpoint `?doc_itself=` пагинирован и параметр `page` игнорирует** —
   отдаёт только первую порцию (`textCompleted='False'`). Полный текст отдаёт только
   экспорт `?savertf=...&page=all` (несмотря на имя — это MHTML с HTML внутри, не RTF).
3. **Составные номера статей — надстрочные**: `Статья 181<span class="W9">1</span>.`
   означает ст. 181.1; бывает и с дефисом: `348<sup>11-1</sup>` = ст. 348.11-1.
   Парсер склеивает это в `181.1` / `348.11-1`.

### Проверка (2026-07-06, ТК РФ)

| Что проверяли | Результат |
|---|---|
| Редакция | rdk=192 от 15.05.2026 (№ 32-П) — последняя из 118 |
| Полнота | **538 статей**, 70 глав; контрольные статьи поздних поправок на месте: 64.1 (2008), 312.1 (дистанционка, 2013), 351.7 (2022) |
| Актуальность формулировок | ст. 144 — «Системы оплаты труда работников государственных и муниципальных учреждений» (редакция 2006+), а не исходная «Стимулирующие выплаты» |
| Утратившие силу | ст. 130, 294, 333 помечены `status=repealed` |
| Разметка | `<p class="T">` — часть/раздел, `<p class="H">` — глава и статья, `<p>` без класса — текст |

### Почему именно этот источник

1. **Официальный** — на защите можно честно сказать «данные из государственной ИПС "Законодательство России"», юридически корректный первоисточник.
2. **Полный консолидированный текст** (все 118 редакций уже слиты) — двумя HTTP-запросами, без обхода пагинации и рендеринга JS.
3. **Стабильная класс-разметка** — парсер простой; на любое изменение разметки отвечает громким `ParseError`, а не молчаливой порчей базы.
4. Бесплатно, без ключей и регистрации.

## Отклонённые варианты

| Источник | Почему нет |
|---|---|
| `publication.pravo.gov.ru` (офиц. опубликование) | отдаёт только **PDF первоначальных публикаций** — без консолидированных редакций; пришлось бы самим накладывать сотни поправок |
| `consultant.ru` | удобная разметка, но каждая статья — отдельная страница (сотни запросов), и условия использования запрещают автоматизированный сбор |
| `garant.ru` | то же: коммерческая СПС, скрапинг против ToS |
| Викитека и зеркала | неофициальные, риск устаревших редакций — недопустимо для юридического бота |

## Известные ограничения (зафиксированы, решение — фаза 2)

- **`effective_date`** в payload — дата последней редакции **документа целиком** (из шага 1), не каждой статьи в отдельности. По-статейные даты требуют разбора сносок — фаза 2 (там же diff-отчёты для пушей).
- Endpoint недокументирован (внутренний API портала) — при поломке разметки парсер падает с `ParseError`.
- `source_url` в цитатах ведёт на документ целиком (анкоров по статьям у ИПС нет).
- Сноски «(В редакции федерального закона …)» вычищаются из текста чанков — они повторяются сотнями и шумят в векторном поиске.

## Как добавить новый акт

`nd` нового кодекса ищется поиском на http://pravo.gov.ru/proxy/ips/ (id виден в URL карточки документа), после чего добавляется одна строка в реестр `pipeline/acts.py`.
