# STATUS — живой журнал команды

> Единственный источник правды о том, кто что сделал. Обновляется каждым из четырёх клаудов.

## Правила (для людей и их клаудов)

1. **В начале сессии**: `git pull origin main`, затем скажи своему Claude прочитать этот файл — он узнает актуальное состояние всех ролей.
2. **В конце каждого рабочего блока**: Claude дописывает запись в **свою** секцию (чужие секции не редактировать — это правило владения из TEAM_PLAN, применённое к файлу).
3. Формат записи: дата → сделано → дальше → «важно другим» (если есть).
4. Коммит этого файла — **можно сразу в main без PR** (единственное исключение из правил веток; файл журнальный, секции не пересекаются). Сообщение коммита: `status: роль N`. Если пуш отбился — `git pull --rebase origin main` и пуш снова.
