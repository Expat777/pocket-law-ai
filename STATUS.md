## Роль 2 · Agent (`agent/`) — ветка `Roman_SPT` · **функционально завершена**
- Граф `intent → retrieve → verify → {compose|clarify|refuse|offtopic}`, `Agent()` drop-in; LLM абстрагирован; бюджеты (max_tokens 900, recursion 8, обрезка 4000). **Anti-hallucination:** цитаты только из verify; `INSUFFICIENT` ловится по границе слова где угодно; промпт-инъекции (файл/URL/социнженерия) проверены на живом LLM.
- **Ретрив-стек:** роутинг **53 ветки** (`BRANCH_TO_ACTS` — единый источник правды, промпт автогенерится, канарейка синхронности); квота top-K на акт + **round-robin** по актам; **keyword-net** (ипотека/осаго/микрозайм/МФО/маткапитал/коллекторы; омоним «коллектор»-сантехника разведён); номерной **fast-path** («ст.158 УК»); **HyDE-фьюжн** (`AGENT_HYDE`, параллельно intent, сбой не роняет ответ); **якоря фундамент-статей** (залив/собака→ГК 1064, ДТП→ГК 1079; каждый пункт валидирован судьёй; `AGENT_ANCHOR_ARTICLES`).
- **Цитаты:** выравнивание «Основание» = {статьи в прозе} ∩ {найдено ретривом} — мусор и негрунтованные номера не попадают; `Citation.text` — дословная выдержка ≤450 симв из ретрив-чанка (grounding, не LLM).
- **Документы:** ingest PDF+OCR(rus) с **гейтом качества** (<4 словоподобных токенов → «пришлите чётче», стоп галлюцинациям на фото) и предобработкой (EXIF/грейскейл/автоконтраст/апскейл 1600px); URL с SSRF-защитой; изоляция по `user_id`; **doc-режим: документ ведёт разбор** (`COMPOSE_DOC_SYSTEM`: что это / что хотят / по закону / что делать / антифрод как ВЕРОЯТНОСТЬ, прямой ответ на «это мошенники?»); разбор идёт по упорядоченной голове документа, не по top-K.
- **STT:** `transcribe_audio` (Whisper через Polza, тот же ключ; ручки `STT_MODEL/STT_LANGUAGE/STT_MAX_BYTES`).
- **LLM-as-Judge** (`agent/eval/`): офлайн-судья (temp=0, рубрика grounded/correct/citations/complete, 0-8), валидирован на эталонах; **отсеял 3 «улучшения»** (чеклист договора — утечка эталона в первом замере; RR-лимит — регресс на обеих группах; глобальное снижение температуры — не помогает) — метрическая дисциплина против иллюзий.
- **Известное:** стабильный промах ТК-81 («уволить в отпуске» — ст.81 не ретривится, семантический разрыв; якорь проверен судьёй и отклонён — модель не называет ст.81 в прозе, расширять метку = подгонка); остальные промахи скачут между прогонами (вариативность композиции ±2); confidence-телеметрия пишется в Postgres на всех юр-исходах.
- **Полный аудит зоны (07-13): 8 находок, все закрыты** (31/31 тестов + live-смоук). Главные: «Основание» теряло ПЕРЕЧИСЛЕНИЯ статей («ст. 115, 133 ТК» ронял 133, «статьями 91 и 92» не матчился) → регэксп на списки/словоформы в compose И fast-path (live: «сравни ст. 91 и 92 ТК» → обе цитаты); `verify_citation` считал живой только `active`, а контракт допускает `amended` (изменена, но В СИЛЕ — латентная тихая потеря норм класса КоАП-12.9 при фазе 2 Роли 3) → active|amended|None; кэш Qdrant-клиентов в lookup/ingest (был новый клиент на каждый вопрос с якорем); `graph.ingest_document` терял `filename` из контракта 3.1; STT-ретраи на 503 (как у LLM); `content:null` → "".
- **«Покусала собака» (репро Роли 4) — закрыто:** keyword-net `собак/выгул` → 498-ФЗ (intent на 53 ветках терял вторичную ветку, фильтр acts отрезал закон до ранжирования — паттерн «ипотеки» из бага A). Live 4/4: покус → ГК 1064 (якорь) + 498-ФЗ ст.21/9; выгул → ст.13 первой.
- **Реальный eval с Pravoved (идея пользователя, 07-13):** собрал 40 живых клиентских вопросов / 18 рубрик (robots-дружелюбно, ПДн-скраб, в репо не кладём; 9111/pravorub не подошли). Прогон агент+судья: **средний 4.4/8, медиана 4** — против ~7/8 на рукописном eval: реальная речь заметно тяжелее. Хорошее: честные отказы на непокрытых темах работают (7 REF: таможня/расчёты/военные выплаты), «книжные» кейсы держатся и в живой речи (4×8/8: завещание, парковка-инвалиды, недееспособность, авто в браке). Честный итог: гигиена + стабильнее трейсы, не лекарство от разброса. 32/32 тестов.

## Роль 3 · Pipeline (`pipeline/`) — ветка `Roma_MSK` · **DoD закрыт** (42 теста)

> **Цифры к защите (57 и 53 — обе верны, единицы разные):** **57** = кодов пайплайна (`--act`) · **53** = уникальных `act` в payload — многочастные кодексы под одним act (ГК ч.1–4 → «ГК РФ», НК ч.1–2 → «НК РФ»). Слайд: **«53 акта (12 кодексов + 41 закон), 57 источников, 15 809 чанков»**.

- **Пайплайн:** `fetch → parse → chunk → embed → upsert`, CLI `python -m pipeline.run --act <code>` (`--force` — перезалить игнорируя state; `--dry-run`). Источник — ИПС pravo.gov.ru, экспорт последней действующей редакции. Чанк = **статья** (длинные режутся по абзацам, ≤3000 симв., заголовок дублируется в каждый чанк); `id = uuid5("{act}:{article}:{i}")` + версии по hash текста → идемпотентно. Обновление акта = delete-by-filter `act==X` + upsert.
- **Хранение:** ОДНА коллекция + роутинг по полю `act` — стыки (ТК+СК) бесплатны, интерференция ничтожна (MRR −0.005). **Бенчмарк (ключ к защите):** bge-m3 R@5 **0.97** книжный / **0.84** живой; reranker (bge-reranker-v2-m3) и гибрид BM25+dense измерены — **оба НЕ помогают** (обоснованный негативный результат). Eval-наборы в `pipeline/`.
- **Победы над источником:** редакция по **макс. дате**, не max(rdk) — rdk не монотонен (иначе залили бы право 2001 г.); фолбэк `savertf`→`fulltext` (НК ч.2 ~9МБ и короткие законы); **ложный `repealed`** («часть утратила силу» гасила всю статью, в КоАП ~49 живых вкл. 12.9) — починен end-to-end (КоАП 88→39, 0 ложных по базе); двойные дефисы `201.15-2-1`, старый формат 1991 г., голые стабы; **коллизии номеров ФЗ** (115-ФЗ и 218-ФЗ — по два разных закона; старые акты — римская цифра `4462-I`; 255-ФЗ прятался под исходным названием) → каждый `nd` подтверждается реальным `fetch+parse`, не выдачей поиска.
