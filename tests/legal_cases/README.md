# tests/legal_cases — эталонные юрвопросы (ручная сверка)

Владелец: **Роль 2**. Назначение — доказать точность агента на защите: единственный
способ проверить, что бот отвечает по делу и цитирует правильные статьи, — прогнать
эти кейсы и глазами сверить ответы (в команде нет юриста, автотест «правильности
права» невозможен).

## Формат `cases.json`

```jsonc
{
  "cases": [
    {
      "id": "dismissal-during-vacation",     // стабильный идентификатор
      "question": "...",                      // как спросит пользователь
      "branch": "трудовое",                   // отрасль (или null)
      "expected": { "act": "ТК РФ", "articles": ["81"] },  // ожидаемые статьи в цитатах
      "note": "..."                            // чем проверяется кейс
    }
  ]
}
```

`expected.articles: []` и `act: null` — контрольные кейсы, где агент НЕ должен
цитировать закон (вопрос вне права → уточнение или отказ).

## Как использовать

Пока LLM и боевые данные не подключены — сверка ручная:

1. Подними Qdrant с ТК РФ (пайплайн Роли 3) и агент с реальным `search_law`.
2. Для каждого кейса вызови `answer_question(user_id, question)`.
3. Сверь: для юр. кейсов — попала ли ожидаемая статья в `answer.citations`;
   для контрольных — сработали ли `refused` / `clarifying_question`.

Автоматический прогон этих кейсов появится, когда подключим LLM-клиент и общий
Qdrant на сервере (тогда добавим скрипт-раннер, печатающий таблицу
«кейс → ожидалось → получено → совпало?»).
