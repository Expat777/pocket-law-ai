# 🏛️ Карманный юрист (pocket-law-ai) — Telegram-бот по законодательству РФ

AI-агент для справок по законодательству РФ с **проверяемыми ссылками на первоисточники**. Пользователь задаёт вопрос или присылает документ (например, договор) — бот отвечает простым языком со ссылкой на конкретную статью нормативного акта, указывает дату редакции и даёт кликабельную ссылку на источник.

> **Дисклеймер.** Проект носит справочный характер и **не заменяет консультацию квалифицированного юриста**. Ответы формируются на основе базы нормативных актов и могут содержать неточности. Для юридически значимых решений обращайтесь к профильному специалисту.

## 🎯 Идея и статус

Помощник по праву РФ, который **не выдумывает нормы, а ссылается на действующий закон**. Первоначально задумывался как офлайн desktop-приложение (Tauri), но для соблюдения срока команда выбрала **Telegram-бот** как основной интерфейс: нулевой онбординг, простая загрузка документов, лёгкая демонстрация.

**Статус:** MVP собран и **живёт в Telegram** ([@pocket_law_ai_bot](https://t.me/pocket_law_ai_bot)) — реальный агент, база в проде, docker-compose на сервере. Учебный/дипломный проект, команда — 4 человека.

**Охват:** база выросла с задуманных «2–3 кодексов» до **35 актов** — 10 кодексов (ТК, УК, ГК ч.1–4, СК, КоАП, ЖК, НК ч.1–2, ГПК, УПК) + ЗоЗПП, ЗК, КАС и ~20 «житейских» ФЗ (потребкредит, ипотека, ДДУ, ОСАГО, банкротство, исполнительное производство, персданные, ЕГРН, пенсии, здоровье/ОМС, образование, миграция, ЗАГС, маткапитал, опека, занятость, приватизация жилья и др.). ~13.8 тыс. проиндексированных фрагментов.

## ✨ Ключевые возможности

- **Вопрос-ответ по праву РФ** простым языком + обязательная ссылка «Основание: ст. N <кодекс> (ред. от …)» с **кликабельным источником** на pravo.gov.ru.
- **Anti-hallucination.** Отдельный узел графа сверяет цитаты с реально найденными фрагментами базы; нет нормы → **честный отказ**, а не выдумка; вопрос вне права → мягкий отказ по области.
- **Уточняющие вопросы.** При недостатке контекста бот переспрашивает, а не додумывает.
- **Мультикодексный роутинг.** Вопрос уходит в нужный акт (КоАП≠КАС, УК≠ГК) — в тестах чистота роутинга 100%.
- **Работа с документами пользователя.** Загрузка PDF/фото (OCR при необходимости) или по URL; ответ строится по содержимому файла. **Мультивыбор** документов (искать по нескольким), авто-скоуп на свежезагруженный, изоляция per-user в Qdrant.
- **Онбординг `/topics`.** Кнопки-темы (трудовые/семейные/потребитель/жильё/штрафы/кредиты) → частые вопросы — снижают порог входа.
- **Экспорт «памятки».** Ответ с источниками можно скачать PDF-файлом (оформлен как справка, не как заключение).
- **Диалоговая память и rate-limit.** История по `telegram_user_id`; лимит запросов на пользователя.
- **HyDE-фьюжн и confidence-телеметрия** на стороне агента (повышение recall, сигнал качества).

**Ключевые архитектурные решения:**

| Решение | Обоснование |
|---|---|
| Telegram-бот вместо desktop | Уложиться в срок, нулевой онбординг, простая загрузка файлов |
| Бот зависит от `AgentClient`-**Protocol**, не от класса `Agent` | Офлайн-тесты бота на `MockAgent`; замена мока на реального агента = одна строка |
| **Одна** коллекция `law_articles` + роутинг по полю `act` | «Не уверен» = поиск по всем (мягкая деградация вместо жёсткого промаха); стыковые вопросы бесплатны |
| Серверный фильтр `search_law(acts, doc_ids)` (MatchAny) | top-K считается ВНУТРИ нужных актов/документов, а не режется по всей базе |
| `verify` перед `compose` | Цитаты только из реально найденных фрагментов — anti-hallucination |
| `insufficient` → уточнение/отказ | Убирает риск выдумывания при неполном контексте |
| Внешний LLM API (не локальная модель) | Качество на юридических формулировках; модель абстрагирована (смена = переменная) |

> Проект **не использует MCP-инструменты** (были в первоначальном замысле) — агент ходит в поиск напрямую через `search_law` из `shared/`.

## 🧠 Как работает агент (LangGraph)

Граф: `intent → retrieve → verify → {compose | clarify | refuse | offtopic}`.

1. **`intent`** — определяет, юридический ли вопрос (`is_legal`), и вероятные отрасли/акты (`BRANCH_TO_ACTS`). Переформулирует «житейский → юридический». Не-право → `offtopic` (мягкий отказ по области).
2. **`retrieve`** — серверный поиск `search_law(query, user_id, acts, doc_ids)` по Qdrant (`MatchAny` по актам/документам). Приёмы точности: квота top-K на кодекс, номерной fast-path («ст. 158 УК»), **HyDE-фьюжн** (гипотетический ответ генерится параллельно intent — латентность не растёт), keyword-net на однозначные термины, round-robin по актам на стыковых вопросах.
3. **`verify`** — сверяет кандидатов-цитаты с найденными фрагментами (существование + актуальность редакции). Защита от галлюцинаций.
4. **Ветвление:** нет норм → `refuse` (честный отказ); мало контекста → `clarify` (уточнение); есть основание → `compose`.
5. **`compose`** — собирает ответ через LLM: текст + цитаты + дисклеймер. При скоупе на документ — **разбор загруженного файла по упорядоченным чанкам** (а не по «похожим» фрагментам).

Параллельно пишется **confidence-телеметрия** на всех юр-исходах (сигнал качества). Загруженный документ извлекается (`ingest_document`: PyMuPDF + OCR) и индексируется в `user_documents` изолированно по `user_id`.

## 🔄 Data Pipeline — база законов

CLI-скрипт с идемпотентными шагами (`python -m pipeline.run --act <code>`):

```
fetch          # выгрузка последней редакции с ИПС pravo.gov.ru (двухшаговый экспорт)
  → parse      # HTML + PDF/OCR, структура статья/пункт, детект утративших силу
  → chunk      # разбиение по смысловым границам
  → embed      # эмбеддинги deepvk/USER-bge-m3 (1024)
  → upsert     # запись в Qdrant (law_articles), версионирование по hash (без дублей)
```

Редакция берётся **по максимальной дате** (пойман баг немонотонного `rdk` — иначе можно залить норму 20-летней давности). `source_url` в payload → кликабельные цитаты. Обновление одного акта не трогает остальные (delete-by-filter `act==X` + upsert). Бенчмарк эмбеддингов обоснован (bge-m3 победила e5; reranker и BM25-гибрид измерены и отклонены как не помогающие).

## 📊 Метрики качества (к защите)

- **Recall@цитаты ТК: 98%** (61/62; HyDE поднял с 95%).
- **Мультикодексный gold-набор: Recall@цитаты 95%** по 10 кодексам + ФЗ.
- **Чистота роутинга (routing purity): 100%** — книжные 28/28 + житейские 35/35; ни одной чужой статьи в цитатах.
- **Житейский recall@ретрив: 91–94%** (вариативность LLM-переформулировки).
- Контроль вне права 1/1; anti-hallucination подтверждён вживую (в т.ч. устойчивость к инъекциям через файл/URL и социнженерию).

## 🔐 Безопасность и приватность

> ⚠️ Бот работает через сервер и принимает документы → де-факто обрабатывает **персональные данные** (152-ФЗ). Реализованы базовые меры; полное соответствие (уведомление РКН, политика ПДн) — вне scope MVP.

- **Согласие на обработку ПДн** при `/start` — без него бот не отвечает (гейт-middleware).
- **Дисклеймер в каждом ответе** («Не является юридической консультацией»).
- **Полное удаление данных** по `/delete`: чистит Postgres (согласие + история) **и** документы пользователя в Qdrant.
- **Валидация файлов по magic bytes** (не по расширению) + лимит размера ДО скачивания.
- **Rate-limit** на пользователя (по умолчанию 20/час), поверх `dialog_history`.
- **Изоляция документов** per-user в Qdrant (фильтр по `user_id`/`doc_id`).
- **Egress до Telegram через xray-proxy** (хостинг режет `api.telegram.org`); LLM/Qdrant доступны напрямую.
- Секреты только на сервере (`.env`, gitignored), не в репозитории.

## 🚀 Установка и запуск


## 🆚 В чём преимущество над обычным ИИ-чатом

Мы **не конкурируем с ChatGPT по «уму» модели** (LLM тот же, облачный). Преимущество — в **заземлённости и проверяемости**:

- **Обязательная ссылка на статью + редакция + кликабельный источник** — можно проверить; голый чат выдумывает номера статей.
- **Актуальность** — база из текущих редакций (2025–2026) с версионированием; у LLM — cutoff обучения.
- **Честный отказ** вместо уверенной выдумки — для права критично.
- **Работа с личным документом** пользователя + мультивыбор, изоляция, удаление (152-ФЗ).
- **Российский контур и приватность** (152-ФЗ), доступность в Telegram без VPN.

## ⚠️ Известные ограничения

- **Юридическая точность.** В команде нет профильного юриста — точность проверяется вручную по открытым источникам. Не полагаться на бота в юридически значимых ситуациях.
- **Точность цитат на новых ФЗ** дошлифовывается: нужный акт находится, но конкретная статья внутри иногда «рядом» (роутинг актов — 100%, precision статьи — в работе).
- **OCR кириллицы.** Tesseract на плохих сканах уступает; для сложных — потенциальный fallback на облачный OCR.
- **Лимит файлов Telegram** — до 20 МБ; многостраничные сканы могут не пройти.
- **Единая точка отказа** — всё на одной VM; критична регулярность бэкапов.
- **Соответствие 152-ФЗ** — реализованы базовые меры, полное соответствие вне scope MVP.
