# pocket-law-ai — правила для Claude Code

Полный контекст проекта — в [TEAM_PLAN.md](TEAM_PLAN.md). Прочитай его целиком перед началом работы.

## Карта владения папками

| Папка | Владелец | Что там |
|---|---|---|
| `bot/` | Роль 1 | aiogram: handlers, FSM, форматирование |
| `agent/` | Роль 2 | LangGraph: граф, инструменты, промпты |
| `pipeline/` | Роль 3 | fetch/parse/chunk/embed/upsert законов |
| `infra/` | Роль 4 | docker-compose, миграции, скрипты |
| `shared/` | Роль 4 | контракты (Pydantic-модели, сигнатуры) |
| `tests/legal_cases/` | Роль 2 | эталонные юрвопросы для ручной сверки |
| корень (`CLAUDE.md`, `README.md`, `.env.example`) | Роль 4 | правила и запуск |

## Главные правила

1. **Никто не редактирует чужие папки.** Нужно изменение у соседа — issue или сообщение владельцу, он делает сам.
2. **`shared/contracts.py` меняется только через PR с тегом всех четверых.** Пока PR не смержен, контракт считается старым.
3. `main` всегда рабочий. Мерж только через PR. Ветки: `dev1/*`, `dev2/*`, `dev3/*`, `dev4/*`.
4. Если два PR трогают один файл — значит, нарушена карта владения; файл отдаётся владельцу.

## Запуск

```bash
cp .env.example .env   # заполнить токены/ключи
make up                # docker-compose up (qdrant + postgres + bot)
make migrate            # применить миграции Postgres
make fixtures           # загрузить фикстуры законов (Роль 2)
make test
```
